# 開始使用

開始學 scpp 的最好方式，和學大多數語言時一樣：先把工具跑起來，編譯一個極小的程
式，再把同樣的事情放進一個更像「專案」的目錄結構裡。

這一章只做三件事：

1. 從原始碼建置編譯器；
2. 編譯並執行一個單檔的 “Hello, world!” 程式；
3. 建立一個最小但確實可用的 manifest-based 專案，並用 `scpp build` 建置它。

你現在還不需要徹底理解所有權、借用或 `[[scpp::unsafe]]`。這一章的目標，只是先讓
一個真正的程式跑到你的螢幕上。後面的章節會再解釋這門語言為什麼這樣設計。
