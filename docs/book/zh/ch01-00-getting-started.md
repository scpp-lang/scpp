# 开始上手

开始学 scpp 的最好方式，和开始学大多数语言时一样：先把工具跑起来，编译一个极小
的程序，再把同样的事情放进一个稍微更像“项目”的目录结构里。

这一章只做三件事：

1. 从源码构建编译器；
2. 编译并运行一个单文件的 “Hello, world!” 程序；
3. 创建一个最小但真实可用的 manifest-based project，并用 `scpp build`
   构建它。

你现在还不需要彻底理解所有权、借用或者 `[[scpp::unsafe]]`。这一章的目标只是先
让一个真实程序跑在你的屏幕上。后面的章节会再解释语言为什么这样设计。
