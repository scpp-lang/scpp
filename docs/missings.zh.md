# 在重写本书时注意到的类 Rust 能力缺口

此文件是内部积压清单，不是面向读者的书籍内容。

## 语言 / 标准库缺口

- 目前还没有类似 `std::string_view` 的借用文本视图，这使得讲解类似 Rust 的“借用文本切片”示例更困难。
- `std::span` 目前只能由定长数组构造，还没有从其他容器类型构造的更广泛路径。
- `std::span` 在构造后不能重新绑定，因此它的行为更像一个永久绑定的借用，而不是一个可以自由重新赋值的视图值。
- 目前还不支持用户自定义 `operator->`：声明 `operator->` 会直接报错
  （`expected ';' but found '->'`）；自定义类型上的 `x->y` 目前只能通过
  `operator*` 实现，这不同于真实 C++ 里独立的 `operator->` 链式 /
  proxy 语义模型。
- 跨函数的具名生命周期组大多仍未实现；今天 `[[scpp::lifetime(...)]]` 这套机制里只有很窄的一小部分可用。
- 还没有 Rust 风格的带负载枚举、模式匹配、`if let` 或 `let...else`；当前的 `enum class` 支持只覆盖 C++ 风格的无字段枚举。
- 还没有像 `std::vector` 或哈希映射等价物这样的可增长标准容器。
- `const double&` 形参绑定仍然拒绝 `double` 字面量，因此希望对浮点字面量调用也友好的 API 目前不得不优先使用按值传参。
- 目前还没有基于虚函数的动态分发，尽管基础的单继承现在已经能工作。
- 还没有 async/await、future 或 stream 相关叙事，所以 TRPL 的 async 章节目前在 scpp 中没有直接对应内容。
- 还没有宏系统，这使得整整一类 Rust 元编程示例无法进入任何近期的 scpp 书籍。
